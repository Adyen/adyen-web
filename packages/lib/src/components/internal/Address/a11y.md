# Address Component — Accessibility Notes

## Search bar DOM order

The search field is rendered in the DOM **before** the "+ Enter manually" button, even if the visual layout suggests otherwise. This is intentional: the primary flow is address lookup, and the manual entry option is a fallback. Placing the search field first in focus order ensures screen reader and keyboard users encounter it first and are not misled into thinking manual entry is their only option.

This was reviewed and confirmed by our accessibility partner:

> According to [WCAG SC 2.4.3 Focus Order](https://www.w3.org/TR/WCAG22/#focus-order) "focus order does not necessarily need to follow the visual presentation or layout" and in this case we agree it makes sense from a user experience standpoint to have the address field receive focus and announce before the "Enter address manually" control. Otherwise, a screen reader user might think that they have to manually enter the address without first having another option.

The associated finding (COW–E77) was dismissed in their platform after this review.
